A long time ago...

The year was 1994, and Netscape came on the scene as one of the first web browsers. A year later, Netscape released version 2.0 which was the first browser to support the Javascript language. First environment, really
The idea of Javascript was to enable webpages to behave dynamically. Prior to Javascript, the web was mostly just hypertext. Netscape grew quickly, and by 1996 it had 80% market share.
Meanwhile, Microsoft was working on a new browser too
The first version of IE was included as a part of the Windows 95 Plus Pack add-on.
This was the first of the so-called browser wars Netscape had 80% of the market and a lots of public goodwill But Microsoft was able to use its heft to win the war. Windows had over 90% share of the desktop operating system market IE eventually overtook Netscape, and in 1998 Netscape was acquired by AOL Aside: Netscape's total revenue never exceeded the interest income generated by Microsoft's cash on hand I remember my dad reading an article in the newspaper about Bill Gates having a net worth of 11 billion dollars. 300 million dollars a day in interest
Shortly before its acquisition by AOL in 1998, Netscape released the source code for its browser and created the Mozilla Organization to coordinate future development of its product.
Yay.
In the year 2000, also known as Y2K, Firefox 1.0 was released. It was a breath of fresh air. Flexibility, Extensibility, Clean UI, Performance, Privacy
Firefox 3.5 was released in June 2009 with the TraceMonkey compiler TraceMonkey was the first JIT compiler for JavaScript Dramatic performance improvements ranging between 20 and 40 times faster than 3
Google Chrome was released in 2008 Now has 62% worldwide usage share of desktop web browsers 50% market share across all platforms combined
Google has a dedicated team building a new Javascript engine called v8
"front-end vs back-end" divide Perl, PHP, ColdFusion, .NET, Django, Ruby on Rails human bottlenecks
node is a javscript runtime based on Chrome's V8 Javascript engine node provides a way to execute javascript code outside of a browser this was key for javascript's growth and adoption "web designers" and "frontend developers" can build servers, and database-backed applications, and microservices, and command-line interfaces. `fs` module for accessing the filestystem `net` module for creating web servers `http` module for talking to the internet `process` module for accessing environment variables `vm` module compiles and runs code within V8 Virtual Machine contexts. more importantly, node enabled automation automated testing with headless browsers: phantomjs, capybara, webdriver asset compilation: code linting, concatentaion, asset minification, etc node now has 3.5 million users userbase is consistently growing at 100%/year for the last three years
- one of the most exciting things about node is not the node runtime itself, but the so-called "userland" that has developed around it. - npm is the node package manager - introduced a simple model for publishing and sharing code - npm init; npm publish;
